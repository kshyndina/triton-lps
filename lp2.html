Wallet history in one call

Single-call API

getTransactionsForAddress replaces the 5-step, 100-request loop with a single call that returns the wallet's full activity, ATAs included. Server-side filters on slot, blockTime, signature, and status, with up to 100 transactions or 1,000 signatures per page and a resumable cursor.

Sub-millisecond head cache for recent data

<1ms reads

Recent slots resolve from an in-memory head cache without a database round-trip. processed commitment supported across all historical methods, same JSON-RPC interface, no client change required. Slot range parameters scope reads for the fastest possible response.

Superior performance across the entire ledger

Up to 38x faster at p50

ClickHouse rewrite of the storage layer, with every table physically sorted for the queries it serves so reads skip irrelevant rows. Purpose-built materialised views, hot-address optimisation for heavy-traffic addresses like USDC, and tiered storage that keeps older epochs fast and cost-efficient.
