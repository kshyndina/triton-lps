Indexes that build themselves from your traffic.

A standalone query tracker watches your traffic and counts how often each filter pattern appears. Once a pattern crosses a threshold, Steamboat creates a tailored PostgreSQL index for it automatically. From that point on, matching queries skip the raw scans entirely.

  • Zero manual setup. No index requests to file.
  • Cross-instance coordination. Patterns aggregated across all API instances.
  • Partial indexes and DISTINCT ON queries natively supported
  • Latest version of each account retained, superseded versions garbage-collected
  • Account state bootstrapped from a Solana snapshot, kept current via gRPC stream

SPL Token and Token-2022 reads, optimised by default.

getTokenAccountsByOwner and getTokenAccountsByDelegate run through denormalised columns purpose-built for token queries. No manual index creation, no waiting for the dynamic indexer to spot the pattern. Token reads are fast from the first query.

  • SPL Token and Token-2022 supported out of the box
  • Denormalised columns for hot-path queries
  • Token-2022 extensions fully indexed
  • Wallet balance lookups in milliseconds, not seconds

memcmp, dataSize, and the filters your app already sends.

Steamboat accepts the same filter syntax as Agave's getProgramAccounts. Any memcmp filter your application sends is eligible for indexing. The auto-indexer handles patterns that recur above a threshold. Less common patterns work the same way but use sequential scans until they cross the threshold.

  • Standard memcmp and dataSize filters supported
  • Any filter your app sends, no whitelist
  • Custom indexes for specific programs available on dedicated endpoints
  • Filter combinations and chained filters work as expected

Runs on its own infrastructure, scales independently.

Steamboat runs separately from the Agave validator. Five components (Ingest, Store, Track, Index, Serve) each scale on their own. That's how custom indexes become affordable on shared endpoints and stay performant as the network grows.

  • Decoupled from Agave's key-value store
  • Ingest layer pulls from Dragon's Mouth gRPC, backfills automatically on dropped slots
  • Store layer in PostgreSQL with dynamic index creation
  • Serve layer is stateless JSON-RPC. Routes standard calls to the right database.
  • Open-source under AGPL coming. Self-host on commodity hardware.

Steamboat FAQ.

Lorem ipsum dolor sit amet, consectetur adipiscing elit. The eight most-asked questions about Steamboat.

Does Steamboat change the RPC API?+
No. It serves the same JSON-RPC methods (getProgramAccounts, getTokenAccountsByOwner, getTokenAccountsByDelegate) with the same request and response format. Your existing code works without changes.
How do I know if my query is hitting an index or doing a full scan?+
If you're on a Triton endpoint and querying a program Steamboat ingests, your queries are routed through Steamboat automatically. The auto-indexer monitors your patterns and creates indexes for recurring filters. Once an index exists, subsequent queries with the same pattern hit it directly.
How is Steamboat priced?+
Same as any standard RPC call: $10 per M requests plus $0.08 per GB of bandwidth. This applies whether your query goes through a custom index or not. No premium for indexed reads.
Why is my gPA still slow for some programs?+
If you're querying a program or filter pattern that hasn't been indexed yet, the query may fall back to a scan. On dedicated endpoints, we can scope your instance to exactly the programs you need.
What about compressed NFTs and Bubblegum?+
Compressed NFT data lives off-chain in transaction logs, not in account state. Steamboat indexes account data, so Bubblegum is handled separately by the DAS API. Also included with your deposit.
Can I create any index I want?+
Any memcmp filter your application sends is eligible for indexing. The auto-indexer handles patterns that recur above a threshold. For less common patterns, reach out and we'll help configure your endpoint.
Will Steamboat be open-sourced?+
Yes, under AGPL as part of the Yellowstone project. Once available, you'll be able to self-host on commodity hardware. A full-chain instance with hundreds of query patterns fits in roughly 700 GB of storage, runs on a single PostgreSQL node for under $400 per month.
Should I use Steamboat or Dragon's Mouth?+
Both, usually. Steamboat serves queries on demand for the current account state. Dragon's Mouth streams updates in real time as state changes. Many apps load the full picture via gPA through Steamboat, then subscribe via Dragon's Mouth to keep it in sync without polling.
