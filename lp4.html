The earliest signal on Solana, transactions before the node executes them.

Intercepts the validator pipeline at the CompletedDataSets stage, bypassing Replay entirely. Reconstructs transactions from raw shreds server-side and streams them over gRPC in p50 around 6.3ms from shred arrival. Exposed as SubscribeDeshred inside Yellowstone. Same client, same auth, new subscription type.

  • p50 around 6.3ms, p90 around 20ms from shred arrival
  • Server-side reconstruction. No raw UDP, no Reed-Solomon decoding.
  • Server-side vote filtering. Votes are most of network traffic, rarely useful for traders.
  • Account filtering on static keys and ALT-loaded addresses
  • Trade-off: pre-execution means no logs, status, balance changes, or CUs. Use Dragon's Mouth for confirmation.
  • For HFT, arbitrage, MEV, market making, copy trading, liquidations

Streaming FAQ.

Lorem ipsum dolor sit amet, consectetur adipiscing elit. The eight most-asked questions about the streaming suite.

Do I need to use all 5 streaming products?+
No. Pick the patterns your app needs. Most teams start with one or two. Dragon's Mouth for real-time, Whirligig for the frontend, Fumarole if completeness matters. Add more later, same auth and credentials.
What's the difference between Dragon's Mouth and Yellowstone?+
Yellowstone is the open-source Geyser plugin we built. Dragon's Mouth is the hosted, managed version of it. We run it for you with multi-region, auth, server-side filtering, and SLAs. Same protocol, less ops.
How is latency that low?+
We co-locate streaming clusters with high-stake validators across 20+ regions and ingest shreds directly from Triton's own validators, Jito, DoubleZero, and Turbine. Updates hit the edge as fast as physically possible. Dedicated streaming clusters mean no contention with general RPC traffic.
What if my client disconnects?+
Dragon's Mouth (v13.1 plus) has built-in auto-reconnect with a around 1,000 slot replay window. Set set_reconnect_config on the client builder. Fumarole goes further. Server-side cursor with 4 day retention means you can be offline for days and still resume cleanly.
Can I stream historical data?+
Yes, Old Faithful streams. Same gRPC interface as Dragon's Mouth, any slot range, any speed. Your client code doesn't change between live and historical.
What's excluded from Triton streams?+
Light Protocol / ZK Compression (compr6CUsB5m2jS4Y3831ztGSTnDpnKJTKS95d64XVq). At peak load it accounted for over 50% of all Geyser traffic, making it impractical to include. Use the Triton-hosted Photon service to query compressed accounts, token balances, validity proofs, and transaction signatures directly.
When should I use Deshred vs Dragon's Mouth?+
Deshred for the earliest signal. Pre-execution intent, p50 around 6.3ms from shred arrival. Dragon's Mouth for confirmed results with full execution context (status, balance changes, logs, inner instructions). Many trading strategies combine both: Deshred for speed, Dragon's Mouth for confirmation.
What's the pricing?+
$0.08 per GB of bandwidth across all streaming services. You only pay for the data sent. Server-side filtering reduces traffic significantly. See the streaming best-practices docs for filter patterns and other ways to keep bandwidth low.
