Powering wallets, explorers, and analytics across Solana since 2021.

Solana history, rebuilt for how developers query it

Hydrant indexes the entire ledger into ClickHouse, sorted for the exact patterns wallets, explorers, and analytics tools depend on.

Wallet history in one call −

getTransactionsForAddress replaces the 5-step, 100-requests loop with a single call that returns the wallet's full activity, ATAs included.

  • Server-side filters on slot, blockTime, signature, and status
  • Up to 100 full transactions or 1,000 signatures per page
  • Resumable cursor for paginated reads
  • tokenAccounts: none | balanceChanged | all controls ATA inclusion
Sub-millisecond head cache for recent data +

Recent slots live in an in-memory head cache, resolving in under a millisecond without a database round-trip.

  • Sub-1ms reads on the most recent slots
  • processed commitment supported by all historical methods
  • Same JSON-RPC method, no client change required
  • Slot range parameters to scope reads for the fastest possible response
Superior performance across the entire ledger +

We rewrote the storage layer on ClickHouse and physically sorted every table for the queries it serves, so that reads skip irrelevant rows entirely.

  • Purpose-built materialised views match common access patterns
  • Hot address optimisation. USDC and other heavy-traffic addresses get dedicated lookup tables
  • Tiered storage keeps older epochs fast and cost-efficient to serve
  • Slot range parameters for narrow lookups, replacing signature-based pagination
Affordable and easy to query across every epoch +

We bill every historical method and epoch at the same rate as a standard RPC call. Developer ergonomics are built into the query interface.

  • Bidirectional sorting (asc or desc) on history reads
  • Pagination cursor keeps subsequent calls clean, no resending filters on every page
  • $0.08 per GB of bandwidth and $10 per million requests, every epoch, every method
  • Available on every subscription, no tier gating

Frequently asked questions

Six things teams ask before they switch over.

Do I need to change my code?+
No. Hydrant serves the same standard Solana JSON-RPC methods with spec-compliant responses. getTransactionsForAddress is a new method you can adopt for single-call wallet history.
Does all history cost the same?+
Yes. Every epoch from genesis to today is priced at $0.08 per GB plus $10 per million requests. No premium tier for older data.
How far back does the data go?+
The complete Solana ledger from genesis. Every epoch, block, and transaction. No truncation.
How fresh is recent data?+
Sub-1ms reads on the most recent slots through the head cache. processed commitment becomes available for historical methods on the same path.
Will Hydrant be open-source?+
Yes, under AGPL [verify: ship date]. Run, audit, and extend it yourself.
What about real-time streams or account reads?+
Different products. Hydrant handles historical queries. For real-time, use Dragon's Mouth (gRPC), Whirligig (WebSocket), Fumarole (persistent), or Deshred (pre-execution intent). For account reads, use Steamboat. All included with your deposit.
